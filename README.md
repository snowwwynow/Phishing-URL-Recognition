# Обнаружение фишинговых URL-адресов с помощью логистической регрессии

> Выпускная квалификационная работа: *Разработка и исследование модели машинного обучения для детектирования фишинговых URL-адресов*

---

## Описание проекта

Данный проект представляет собой программную реализацию модели машинного обучения на основе **логистической регрессии**, предназначенной для автоматического обнаружения фишинговых URL-адресов. Модель анализирует набор признаков, извлекаемых из структуры ссылки, и принимает решение о её безопасности.

## ⚙️ Как это работает

### 1. Обучение модели (`training_model.py`)

Скрипт выполняет следующие шаги:
- Загрузка датасета `training_dataset.xlsx`
- Разделение данных на обучающую и тестовую выборки (`train_test_split`)
- Обучение модели **логистической регрессии** (`LogisticRegression`)
- Расчёт метрик качества и построение матрицы ошибок
- Сохранение модели в `features/model/phishing_model.pkl` с помощью `joblib.dump()`

### 2. Предсказание для нового URL (`predict.py`)

Алгоритм работы:
1. Пользователь вводит URL-адрес
2. Проверка на команду выхода (`"стоп"`)
3. Проверка на пустую строку
4. Валидация наличия протокола (`http://` или `https://`) и точки в домене
5. Извлечение признаков с помощью модуля `extract_feat.py`
6. Преобразование признаков в формат `DataFrame`
7. Загрузка сохранённой модели `.pkl`
8. Расчёт вероятности фишинга с порогом классификации **0.3**
9. Вывод результата: *«Ссылка не безопасна»* или *«Ссылка безопасна»*

## 📊 Метрики качества

### До изменения порога классификации (порог = 0.5)

| Метрика | Значение |
|---------|----------|
| **Accuracy** | ≈ 94% |
| **Precision** | ≈ 99% |
| **Recall** | ≈ 88% |
| **F₁-мера** | 0.9305 |

**Проблема:** Recall показал значение 88%, что означает, что модель пропускает 12% фишинговых URL-адресов. В задачах информационной безопасности это недопустимо, так как каждая пропущенная угроза может привести к серьёзным последствиям.

---

### После изменения порога классификации (порог = 0.3)

| Метрика | Значение |
|---------|----------|
| **Accuracy** | — |
| **Precision** | ≈ 94% |
| **Recall** | ≈ 92% |

**Результат:** Как и ожидалось, Precision снизился с 99% до 94%, зато Recall повысился с 88% до 92%. Это сделало модель более сбалансированной и устойчивой к фишинговым атакам. Небольшое снижение Precision означает, что модель теперь может ошибочно заблокировать немного больше безопасных сайтов, но при этом значительно лучше обнаруживает реальные угрозы.

## 📈 Матрица ошибок

|            | Предсказано 0 | Предсказано 1 |
|------------|--------------|--------------|
| **Реально 0** | 100          | 12           |
| **Реально 1** | 1            | 87           |

- **True Negative (TN) = 100** — правильно предсказаны безопасные сайты
- **False Positive (FP) = 12** — ошибочно заблокированы безопасные сайты
- **False Negative (FN) = 1** — пропущены фишинговые сайты
- **True Positive (TP) = 87** — правильно обнаружены фишинговые сайты
